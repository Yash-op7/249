# **Topic 4: Data Analytics Languages (Python & R)**

_(MCQ-focused, practical understanding — not coding-heavy)_

---

## **Term 1: Python Basics**

1. **Interpreted language**:
    
    - Executes line by line (no compilation step).
        
2. **Dynamically typed**:
    
    - Variable type decided at runtime.
        
3. **Indentation-based syntax**:
    
    - Indentation defines code blocks.
        
4. **High-level language**:
    
    - Hides memory management.
        
5. **Platform independent**.
    
6. **Used for**:
    
    - Data analysis, ML, scripting, web.
        
7. **Case-sensitive**.
    
8. **Variables**:
    
    - No declaration needed.
        
9. **Garbage collection**:
    
    - Automatic memory management.
        
10. **MCQ trap**:
    
    - Python is NOT statically typed.
        

---

## **Term 2: Python Data Types**

1. **int**:
    
    - Whole numbers.
        
2. **float**:
    
    - Decimal numbers.
        
3. **str**:
    
    - Text data (immutable).
        
4. **bool**:
    
    - True / False.
        
5. **list**:
    
    - Ordered, mutable.
        
6. **tuple**:
    
    - Ordered, immutable.
        
7. **set**:
    
    - Unordered, unique elements.
        
8. **dict**:
    
    - Key–value pairs.
        
9. **None**:
    
    - Represents no value.
        
10. **MCQ favorite**:
    
    - List vs Tuple difference.
        

---

## **Term 3: Python Lists**

1. **Ordered collection**.
    
2. **Mutable**:
    
    - Elements can change.
        
3. **Allows duplicates**.
    
4. **Indexing starts at 0**.
    
5. **Supports slicing**.
    
6. **Nested lists allowed**.
    
7. **Common operations**:
    
    - append, remove, pop.
        
8. **Heterogeneous data allowed**.
    
9. **Dynamic size**.
    
10. **MCQ trap**:
    
    - Lists are not fixed-size arrays.
        

---

## **Term 4: Python Dictionaries**

1. **Key–value storage**.
    
2. **Keys must be unique**.
    
3. **Keys are immutable**.
    
4. **Values can be duplicated**.
    
5. **Fast lookup (hash-based)**.
    
6. **Unordered (logically)**.
    
7. **Mutable**.
    
8. **Used for mappings**.
    
9. **Access via keys, not index**.
    
10. **MCQ favorite**:
    
    - dict lookup is O(1) average.
        

---

## **Term 5: Python Functions**

1. **Defined using `def`**.
    
2. **Can return multiple values**.
    
3. **Arguments passed by object reference**.
    
4. **Default arguments supported**.
    
5. **Keyword arguments allowed**.
    
6. **Anonymous functions**:
    
    - lambda.
        
7. **Local vs global variables**.
    
8. **Docstrings**:
    
    - Documentation inside function.
        
9. **Recursive functions allowed**.
    
10. **MCQ trap**:
    
    - Python has no function overloading by signature.

## **Term 7: R Language Basics**

1. **Statistical programming language**.
    
2. **Interpreted**.
    
3. **Used heavily in data analysis**.
    
4. **Vector-based operations**.
    
5. **Case-sensitive**.
    
6. **Strong statistical libraries**.
    
7. **Open source**.
    
8. **Memory-based processing**.
    
9. **Command-line & scripts**.
    
10. **MCQ trap**:

    - Decimal values.
        
2. **Integer**:

3. **Character**:

## **Term 9: Vectors in R**

1. **Basic data structure**.
    
2. **Homogeneous**:
    
    - Same type elements.
        
3. **Created using `c()`**.
    
4. **Indexing starts at 1**.
    
5. **Supports vectorized operations**.
    
6. **Arithmetic is element-wise**.
    
7. **Recycling rule**:
    
    - Short vector reused.
        
8. **Logical indexing allowed**.
    
9. **Efficient computation**.
    
10. **MCQ trap**:
    
    - Vector ≠ list.
        

---

## **Term 10: Data Frames in R**

1. **Tabular data structure**.
    
2. **Columns can be different types**.
    
3. **Similar to SQL table**.
    
4. **Rows represent observations**.
    
5. **Columns represent variables**.
    
6. **Access using `$` or index**.
    
7. **Used in most analyses**.
    
8. **Supports NA values**.
    
9. **Built on lists**.
    
10. **MCQ favorite**:
    
    - Data frame allows heterogeneous columns.
        

---

## **Term 11: NA Values**

1. **Represents missing data**.
    
2. **Different from 0**.
    
3. **Different from NULL**.
    
4. **Operations with NA → NA**.
    
5. **Handled using functions**.
    
6. **Important in analytics**.
    
7. **Python: NaN**.
    
8. **R: NA**.
    
9. **Must be cleaned**.
    
10. **MCQ trap**:
    
    - NA ≠ NULL.
