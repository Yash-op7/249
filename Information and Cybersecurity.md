# **Topic 7: Information & Cyber Security Concepts**

---

## **1. Cyber Attacks**

1. **Cyber attack**: Malicious attempt to damage, disrupt, or gain unauthorized access.
    
2. **Malware**:
    
    - Malicious software.
        
3. **Virus**:
    
    - Attaches to files, needs user action.
        
4. **Worm**:
    
    - Self-replicates, no user action.
        
5. **Trojan Horse**:
    
    - Disguised as legitimate software.
        
6. **Phishing**:
    
    - Fake emails/websites to steal credentials.
        
7. **Ransomware**:
    
    - Encrypts data, demands ransom.
        
8. **DoS attack**:
    
    - Makes service unavailable.
        
9. **DDoS attack**:
    
    - Distributed DoS using many machines.
        
10. **MCQ favorite**:
    
    - Virus ≠ worm ≠ trojan.
        

---

## **2. Software Development Security**

1. Security integrated **during development**, not after.
    
2. **Secure coding practices** reduce vulnerabilities.
    
3. Input validation prevents injection attacks.
    
4. Least privilege principle.
    
5. Code reviews for security flaws.
    
6. Use of secure libraries.
    
7. Patch management important.
    
8. Avoid hardcoded credentials.
    
9. Threat modeling during design.
    
10. **MCQ trap**:
    
    - Security ≠ only testing phase.
        

---

## **3. Network Security**

1. Protects network infrastructure.
    
2. Prevents unauthorized access.
    
3. Uses firewalls, IDS, IPS.
    
4. Encryption protects data in transit.
    
5. Secure protocols preferred (HTTPS, SSH).
    
6. Segmentation reduces attack spread.
    
7. VPN for secure remote access.
    
8. Monitors network traffic.
    
9. Controls inbound/outbound traffic.
    
10. **MCQ favorite**:
    
    - Network security ≠ application security.
        

---

## **4. Authentication**

1. Verifies **identity of user**.
    
2. Different from authorization.
    
3. Types:
    
    - Something you know (password)
        
    - Something you have (OTP)
        
    - Something you are (biometrics)
        
4. Single-factor vs multi-factor authentication.
    
5. MFA increases security.
    
6. Passwords should be hashed.
    
7. Avoid plaintext storage.
    
8. Session-based authentication common.
    
9. Certificates used in systems.
    
10. **MCQ trap**:
    
    - Authentication ≠ authorization.
        

---

## **5. CIA Triad**

### **Confidentiality**

1. Prevents unauthorized data access.
    
2. Achieved using encryption.
    
3. Access control important.
    
4. Applies to sensitive data.
    
5. Breach → data leak.
    

### **Integrity**

6. Ensures data is not altered.
    
7. Hashing ensures integrity.
    
8. Detects tampering.
    
9. Unauthorized modification is violation.
    

### **Availability**

10. Ensures systems are accessible when needed.
    

**MCQ favorite**:

- Match attack → CIA property violated.
    

---

## **6. Network Audit**

1. Systematic examination of network.
    
2. Ensures security policies followed.
    
3. Checks firewall rules.
    
4. Reviews access controls.
    
5. Identifies vulnerabilities.
    
6. Reviews logs.
    
7. Ensures compliance.
    
8. Improves security posture.
    
9. Periodic process.
    
10. **MCQ trap**:
    
    - Audit ≠ attack prevention.
        

---

## **7. Systems Audit**

1. Evaluation of information systems.
    
2. Checks hardware, software, data.
    
3. Ensures system reliability.
    
4. Ensures data integrity.
    
5. Reviews controls and procedures.
    
6. Identifies risks.
    
7. Checks compliance.
    
8. Ensures business continuity.
    
9. Broader than network audit.
    
10. **MCQ favorite**:
    
    - System audit covers entire IT system.
        

---

## **8. Common Security Terms (Rapid MCQ Area)**

1. **Encryption**:
    
    - Converts plaintext to ciphertext.
        
2. **Decryption**:
    
    - Reverse of encryption.
        
3. **Hashing**:
    
    - One-way function.
        
4. **Digital Signature**:
    
    - Ensures authenticity + integrity.
        
5. **Firewall**:
    
    - Filters traffic.
        
6. **Antivirus**:
    
    - Detects malware.
        
7. **IDS**:
    
    - Detects intrusion.
        
8. **IPS**:
    
    - Detects + prevents intrusion.
        
9. **Vulnerability**:
    
    - Weakness.
        
10. **Threat**:
    
    - Potential danger.
